# ol_sg_lasso Virtual vs Physical Expansion Benchmark Report

## Overview

Benchmarks comparing three overlapping SG-Lasso solver implementations:

- **ol_sg_lasso virtual**: New solver using gather/scatter through field index on the physical m x n matrix. Coefficient vector is F-sized (one slot per feature per group membership). No extra matrix memory.
- **ol_sg_lasso physical**: Same solver but builds the full m x F expanded matrix in memory and uses standard BLAS directly. Benchmark-only mode for quantifying the virtual expansion tradeoff.
- **olsg_lasso_logisticr (dual)**: Existing dual-variable overlapping solver with per-unique-feature proximal operator.

All runs: `OPENBLAS_NUM_THREADS=1`, `--lambda 0.5 0.5`, single-threaded solver.

## 0% Overlap Parity

On non-overlapping input (identity field, F = n), `ol_sg_lasso` virtual and physical both produce **byte-identical** `weights.txt` output compared to the non-overlapping `sg_lasso` solver. Same iteration count, same intercept, same weights. This confirms the IEEE 754 identity property (0 + x = x) holds through the gather/scatter path.

## Key Findings

1. **Virtual expansion is strictly faster than physical expansion** across all tested configurations (0.42x--0.86x of physical time). The advantage grows with overlap percentage.

2. **Virtual matvec time is constant with respect to overlap.** It always operates on the fixed n-column physical matrix. Physical matvec scales linearly with F. On the wide dataset at 100% overlap, virtual matvec is ~12.5s vs physical ~25s.

3. **Zero memory overhead for virtual expansion.** Physical doubles the matrix memory at 100% overlap (+2.2 GB on full-scale datasets). Virtual uses only two n-length temp buffers (~48 MB at n=6.2M, FP32).

4. **Virtual beats the dual-variable solver on wide data** (31s vs 52s at 100% overlap with 10k genes). The non-overlapping proximal operators (altra_inplace, treeNorm_flat) operating in expanded space are faster than the dual-variable proximal operator.

5. **Dual-variable solver wins on tall data** (2.1s vs 13.7s at 100% overlap with m=7164). When m >> n, the dual solver's per-iteration cost is lower because it operates in the n-dimensional dual space.

6. **Conclusion:** Physical expansion offers no advantage over virtual expansion. Virtual is faster, uses less memory, and produces the same mathematical result. The physical path served its purpose as a benchmark reference and can be removed.
