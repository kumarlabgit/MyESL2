# v0.2.2 — 2026-05-07

## Breaking — lambda values must be in `(0,1)` for non-PSC paths
`train`, `drphylo`, `aim`, and adaptive sparsification now reject any lambda value outside the open interval `(0,1)` — including the boundary values `0` and `1`. The check runs on every value from `--lambda`, `--lambda-file`, and `--lambda-grid` before any solving begins. Previously-tolerated out-of-range values (which produced degenerate runs anyway) now error with the offending value in the message. PSC is unaffected.

## Breaking — PSC `--auto-pairs-tree` disabled at the CLI
`--auto-pairs-tree` and its tuning flags (`--auto-pairs-method`, `--auto-pairs-num-alternates`, `--auto-pairs-max-combinations`) are no longer accepted. PSC contrasts must come from `--species-groups`, `--response-file`, or `--response-dir`. The implementation is intact; only the CLI surface is gated.

## New — `--use-logspace` for `train`, `drphylo`, `aim`, and adaptive
When set, each `--lambda-grid` linear sweep is replaced with a log-spaced grid:
`λ_i = vmin × (vmax_eff / vmin)^(i/(N-1))`,
where `vmax_eff` is the largest sweep value strictly `< vmax` and `< 1`. Sampling density still comes from your `step`. Applies only to `--lambda-grid` — `--lambda` and `--lambda-file` pass through verbatim. The anchor semantics differ intentionally from PSC's `--use-logspace`, which anchors to `--initial-lambda*`/`--final-lambda*`.

## New — shared group-penalty system in `train`, `drphylo`, `aim`
Four flags (previously PSC-only) are now accepted:
- `--group-penalty-type std|sqrt|linear|median` — default `std` = `sqrt(feature_count)`, which preserves prior behavior
- `--initial-gp-value`, `--final-gp-value`, `--gp-step` — drive a penalty sweep when type is `linear`

**Output layout change** when `--group-penalty-type` is anything other than `std`, or when the sweep produces more than one penalty term: lambda subdirs are nested one level deeper under `penalty_<N>/`, and `gss_median.txt`, `bss_median.txt`, `pss_median.txt` move inside each penalty dir. Scripts that hard-code `output_dir/lambda_N/...` paths need to handle the extra layer in those modes. Default behavior (`std`, single penalty) is unchanged.

## New — PSC `--alignments-list`
Accepts a list file specifying overlapping groups of alignments — one group per line, comma-separated paths relative to `alignments_dir`. The legacy directory-enumeration path runs byte-for-byte unchanged when the flag is absent.

- Empty groups are dropped; groups with identical membership are deduplicated (first occurrence wins, drop counts logged).
- Any group with more than one entry requires `--method olsg_lasso_logisticr` or `olsg_lasso_leastr`; otherwise the run errors before any computation.
- The resolved field array is written to `<output_dir>/<base>_field.csv` and injected into the solver via `params["field"]` (does not overwrite a user-supplied `--param field=`).

## New — PSC `--dump-weights`
Opt-in. Writes the full feature-weight vector from every solver call to:

```
<output_dir>/[combo_N/][penalty_P/]lambda_L/weights.tsv
```

The `combo_N/` and `penalty_P/` layers are skipped when there's only one combo or one penalty term, mirroring `train`'s nesting. Under `--make-pair-randomized-null-models` the combo dir name carries a `_rep_<n>` suffix.

File format: 2-column TSV with header. First data row is `__intercept__\t<intercept>`; remaining rows are non-zero feature weights only, using the same `gene_position_aa` labels as the selected-sites CSV. Numbers are `std::scientific` with 10-digit precision (round-trips a double). Safe under `--threads N` (verified byte-identical between `--threads 1` and `--threads 4`).

## Bug fix
PSC `--maxiter` now propagates into the solver's params map even when `--param maxIter=` is not passed explicitly. Previously the flag was silently ignored unless echoed via `--param`.
