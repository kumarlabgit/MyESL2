# v0.2.0 — 2026-05-04

## Performance
- ~2× speedup on the sparse-group-lasso solver for tall matrices (Ukraine benchmark, m=1433, n≈79k: 239s → 114s), and a smaller improvement on wide matrices (DrPhylo, m=97, n≈2.5M: 807s → 722s). All 8 sg_lasso and overlapping-sg-lasso variants now delegate matvec to OpenBLAS (`cblas_dgemv`/`cblas_sgemv`) instead of using the prior hand-written loops.

## Breaking — `drphylo` CLI rework
Tree mode now requires `--tree`, and direct mode takes 3 positional args matching `train`/`aim`:
- Direct: `drphylo <list> <hypothesis> <output_dir>`
- Tree:   `drphylo <list> <output_dir> --tree <tree.nwk>`

The old `drphylo <list> <tree.nwk> <output_dir>` invocation no longer works. Passing `--clade-list` without `--tree` raises a clear error early.

## Breaking — stricter input validation
- Hypothesis files: labels must be `-1`, `0`, or `1`; non-numeric labels and trailing junk are rejected with line numbers.
- Each hypothesis must have at least 2 non-zero samples.
- After encoding, features must span at least 2 distinct alignment positions.

Files that previously slipped through silently can now error.

## Behavior change — `--minor-column` / `--tiered-minor-col`
Emit one column per qualifying alignment position (e.g. `{stem}_{pos}_minor`, `{stem}_{pos}_tminor_Xpct`) instead of a single summary column per gene. Column counts grow accordingly. **Reverted in v0.2.1.**
