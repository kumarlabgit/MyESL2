# v0.2.1 — 2026-05-06

## Breaking — `--minor-column` reverted to per-gene
Emits one summary column per gene, not per position. If you depended on the per-position columns introduced in v0.2.0, your column layout changes back.

## New — `--tiered-minor-col` (per-gene)
Per gene, produces up to four binary columns at frequency tiers 0%, 0.1%, 1%, and 5% (`{stem}_tminor_0pct`, `..._0p1pct`, `..._1pct`, `..._5pct`). Each column is `1` if the species has any minor allele at any position in the gene whose per-position frequency exceeds the tier threshold. Mutually exclusive with `--minor-column`.

Writes a new `tiered_minor_alleles.txt` (stem, pos, allele, freq) that `evaluate` consumes via `--tiered-minor-alleles <file>` to reconstitute the columns for new species.

## Documentation
README and `CLAUDE.md` catch up with the drphylo CLI change and document `--precision`, `--max-mem`, `--adaptive-sparsification`, `--tiered-minor-col`, `--tiered-minor-alleles`, `--gene-limit`, `--species-limit`, and the new `tiered_minor_alleles.txt` output.
